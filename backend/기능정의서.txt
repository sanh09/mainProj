CanSi 기능 정의서 (업데이트 2026-02-12)

1. 목적
계약서(PDF/이미지)를 입력받아 조항을 추출하고 위험 조항을 분류/설명하며, 관련 판례/법령과 토론 요약을 제공한다.

2. 범위
- 대상: 백엔드 파이프라인 + FastAPI API
- 입력: 계약서 PDF/이미지 파일 (+ 선택적 user_id/email, original_name)
- 출력: 조항 목록, 위험 조항, 관련 판례/법령, 토론/요약 결과, 계약 유형(contract_type)

3. 파이프라인 흐름(현재 구현 기준)
1) OCR (Upstage)로 텍스트 추출
2) 텍스트 정제 및 조항 분리 (규칙 기반, 실패 시 LLM fallback)
3) 위험 조항 평가 (OpenAI LLM, 병렬)
4) 판례/법령 수집 (law.go.kr DRF API, DB 캐시 우선, 부족 시 fallback 검색)
5) 임베딩 생성 및 유사도 매칭 (OpenAI Embedding 또는 DB 벡터 검색)
6) 위험 유형 매핑 (키워드 기반)
7) 계약 유형 감지 (jeonse/monthly_rent/sale/lease 등)
8) 토론/요약 생성 (요청 시 또는 백그라운드)

4. 모듈별 기능 요약
4.1 ocr.py
- Upstage OCR 텍스트 추출
- Document Parse HTML 추출 지원 (table-preserving)
- 입력: file/url/base64
- API 키 미설정 시 "api필요" 반환

4.2 text_processor.py
- 공백/노이즈 정리
- 조항 분리: "제n조" 패턴 기반
- 분리 실패 시 LLM(JSON) 기반 fallback

4.3 risk_assessor.py
- LLM 기반 위험도 평가 (low/medium/high/critical)
- 병렬 워커 수: RISK_ASSESSOR_WORKERS

4.4 precedent_fetcher.py / law_fetcher.py
- law.go.kr DRF API 호출
- PRECEDENT_PREFER_DB / LAW_PREFER_DB 설정 시 로컬 DB 캐시 우선
- DB_ONLY 옵션 지원
- 최소 결과 미달 시 조항 제목으로 fallback 검색
- 상세 본문/요약은 detail_limit 내에서 추가 조회

4.5 embedding_manager.py
- OpenAI Embedding 생성
- 조항 ↔ 판례/법령 유사도 계산 후 Top-K 매칭
- USE_DB_VECTOR_SEARCH=true 설정 시 DB 벡터 검색 사용

4.6 risk_mapper.py
- 키워드 기반 위험 카테고리 매핑
- 카테고리 예: 일방적 해지, 무제한 배상, 책임_면책 등

4.7 debate_agents.py
- 임대인/임차인 변호사 + 판사(중재자) 토론 생성
- 판례/법령 근거 인용 라인 생성
- DEBATE_MAX_ROUNDS / DEBATE_MIN_COMMON_POINTS / DEBATE_COMMON_RATIO 등으로 종료 조건 제어

4.8 llm_summarizer.py
- 계약 요약, 종합 리포트, 토론 요약 생성
- clause UI payload(JSON) 생성

4.9 app/main.py (FastAPI)
- 분석 결과 인메모리 캐시(ANALYSIS_STORE) + TTL 관리
- Supabase Storage 업로드(옵션)
- DB 기록: user_files, analysis_history
- 토론/요약 백그라운드 생성 지원

4.10 law_store.py / precedent_store.py
- Postgres + pgvector 기반 캐시 및 벡터 검색
- 키워드 매핑 테이블 관리

5. 결과 데이터 구조 (요약)
5.1 ContractAnalysisResult
- filename
- raw_text
- clauses (Clause[])
- risky_clauses (Clause[])
- precedents (Precedent[])
- laws (Law[])
- raw_html (옵션, document-parse 결과)
- llm_summary (토론 종합 요약 또는 요약)
- debate_transcript (옵션)
- contract_type

5.2 Clause
- id, article_num, title, content
- risk_level, risk_reason
- related_precedents, related_laws

5.3 Precedent / Law
- 판례: case_id, court, date, case_name, summary, key_paragraph
- 법령: doc_id, doc_type, title, summary, content, date, org, url

6. API (FastAPI)
- GET / : health message
- GET /health
- POST /analyze/file : 계약서 분석
  - form-data: file, user_id(옵션), email(옵션), original_name(옵션)
  - 응답: analysis_id, raw_text, clauses, risky_clauses, summary, risk_level 등
  - user_id/email 제공 시 DB 기록 + Supabase 업로드 시도
- GET /history : 사용자 분석 이력(user_id)
- GET /analysis/{analysis_id} : 분석 상세(clauses/risky_clauses/raw_text 포함)
- GET /analysis/{analysis_id}/clause/{clause_id} : 조항 상세 + 토론 요약/소스
- GET /analysis/{analysis_id}/clauses/{clause_id}/debate/summary
- GET /analysis/{analysis_id}/clauses/{clause_id}/debate/transcript
- GET /files : 업로드 파일 목록(user_id)
- POST /signup / POST /login / GET /profile / PUT /profile

7. 환경변수(주요 항목)
- OCR: UPSTAGE_API_KEY, UPSTAGE_OCR_URL, UPSTAGE_DOC_PARSE_URL, UPSTAGE_DOC_PARSE_MODEL, UPSTAGE_DOC_PARSE_MODE
- OpenAI: OPENAI_API_KEY, OPENAI_RISK_MODEL, OPENAI_SUMMARY_MODEL, OPENAI_DEBATE_MODEL, OPENAI_EMBEDDING_MODEL
- 판례/법령: PRECEDENT_API_URL / PRECEDENT_API_KEY, LAW_API_URL / LAW_API_KEY / LAW_TARGETS
- 검색/필터: LAW_DOMAIN_KEYWORDS / LAW_TITLE_MUST_KEYWORDS / LAW_BASE_QUERY
- 상세/페이징: LAW_DETAIL_LIMIT / LAW_DETAIL_TEXT_LIMIT / LAW_PAGE_SIZE / LAW_MAX_PAGES
             PRECEDENT_DETAIL_LIMIT / PRECEDENT_PAGE_SIZE / PRECEDENT_MAX_PAGES
- 캐시/벡터: PRECEDENT_PREFER_DB / LAW_PREFER_DB / USE_DB_VECTOR_SEARCH / EMBEDDING_DIM
- DB-only: PRECEDENT_DB_ONLY / LAW_DB_ONLY / PRECEDENT_DB_LIMIT / LAW_DB_LIMIT
- 토론: DEBATE_BACKGROUND_ENABLED / DEBATE_BACKGROUND_MAX_CLAUSES / DEBATE_BACKGROUND_SLEEP_SEC
        DEBATE_MAX_ROUNDS / DEBATE_MIN_COMMON_POINTS / DEBATE_COMMON_RATIO
        DEBATE_REF_TOP_K / DEBATE_REF_SNIPPET_LEN
- API/스토리지: ANALYSIS_TTL_SECONDS, UPLOAD_DIR, SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, SUPABASE_STORAGE_BUCKET
- DB: DB_HOST / DB_PORT / DB_USER / DB_PASSWORD / DB_NAME / DB_SSLMODE / DB_SCHEMA

8. 제한 사항
- OCR 품질에 따라 조항 분리 정확도 변동
- OCR/LLM/Embedding/판례·법령 API 키 미설정 시 기능 제한
- 외부 API 응답/속도 및 레이트리밋 영향
- 토론/요약은 요청 시 또는 백그라운드에서 생성되며 즉시 반영되지 않을 수 있음
